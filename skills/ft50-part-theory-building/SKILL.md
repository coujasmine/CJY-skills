# Part: Theory Building

## 目标
把"变量关系"升级为"机制化理论命题 + 明确边界"。完成诊断后，直接输出符合 FT50 标准的英文改写版本。

---

## Phase 1: 诊断评审

### 执行步骤
1. 明确刺痛点：现象挑战了哪条既有理论假设。
2. 定义关键概念：工作定义、与近邻构念差异、不可替代性。
3. 搭建机制链：情境 → 机制 → 行为/结构 → 结果。
4. 声明边界：情境、层级、时间、规范目标。
5. 标注贡献类型：概念澄清/新机制/边界扩展/规范重构等。

### 快速判定
- 若"旧概念+新前缀"且理论结构未变：判概念包装。
- 若仅报告中介调节而无理论过程：判弱建构。

- 新命题（P1-Pn）：

- 红线检查：通过/不通过 + 原因

## Phase 2: 英文润色改写

### 强制规则
1. 最终改写内容必须是英文；不输出中文句子作为正文。
2. 不改变原始命题方向与核心机制逻辑，只改语言表达与论证严密度。
3. 命题（Proposition/Hypothesis）必须包含主效应方向、机制路径和边界条件。
4. 每段保留"topic sentence → evidence/logic → implication"结构。

### Theory Building 专用句式模板
#### 理论张力
- `Although [existing theory] posits [assumption], this logic breaks down when [phenomenon/context], because [reason].`
- `The prevailing view that [X] has been challenged by emerging evidence showing [Y].`

#### 命题推导
- `We propose that [X] influences [Y] through [M], particularly when [boundary condition].`
- `Building on this logic, we hypothesize that [specific directional claim] (Hypothesis N).`
- `The underlying mechanism operates because [micro-level logic], which in turn shapes [higher-level outcome].`

#### 边界条件
- `This relationship is contingent on [moderator], such that [direction of moderation].`
- `We expect the effect to be stronger (weaker) when [condition], because [theoretical reasoning].`

#### 概念定义
- `We define [construct] as [concise definition], distinguishing it from [neighboring construct] in that [key differentiator].`
- `Unlike [related concept], [new construct] captures [unique theoretical dimension].`

### 高频误用替换（Theory Building 场景）
- Bad: `X affects Y.` → Better: `X influences Y through [mechanism], particularly when [boundary].`
- Bad: `Based on the above, we propose H1.` → Better: `Building on this reasoning, we hypothesize that [directional claim] (H1).`
- Bad: `The moderating role of Z.` → Better: `We expect Z to strengthen (weaken) this relationship because [theoretical logic].`
- Bad: `We define X as something similar to Y.` → Better: `We define X as [definition], distinguishing it from Y in that [differentiator].`

### 执行流程
1. 基于 Phase 1 诊断结果，确定需要补强的理论建构薄弱点。
2. 建立术语表：统一构念名称、机制术语、边界条件表述。
3. 按"张力 → 概念定义 → 机制推导 → 命题 → 边界"顺序逐段改写。
4. 先做句级修复（语法、逻辑精确性），再做段级重写（推导衔接）。
5. 确保每条命题都包含方向、机制和边界。

### 润色输出模板
1. `Polished Version`（英文，按段输出完整改写的 Theory Building 部分）
2. `Key Edits`（英文，3-8条，说明主要改动与升级点）
3. `Word/Phrase Upgrades`（英文替换对照表）
